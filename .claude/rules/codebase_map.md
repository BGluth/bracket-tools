# Codebase Map

Structural reference for orienting in the mono-repo. No line numbers (they drift); use file paths + type names, then Read to get details.

## bracket-tools-core

Normalized, platform-agnostic domain types.

**Key files:**
- `crates/bracket-tools-core/src/types.rs` — Dehydrateable, Normalizable traits
- `crates/bracket-tools-core/src/data_types.rs` — Id newtypes (TournamentId, BracketId, SetId, GameId, PlayerId), domain structs (Tournament, Bracket, Set, Game, Player), enums (BracketType, GameType, GameWinningSide)

## bracket-tools-cache

Async storage abstraction for caching API responses.

**Key files:**
- `crates/bracket-tools-cache/src/storage.rs` — `Storage` trait (async get/put/delete/clear, `&str` keys, `(SystemTime, Vec<u8>)` values), `StorageError`
- `crates/bracket-tools-cache/src/null_storage.rs` — `NullStorage` (always misses, no-op writes)
- `crates/bracket-tools-cache/src/sled_storage.rs` — `SledStorage` (sled `Db` in `Mutex`, bincode serialization), `SledStorageBuilder`
- `crates/bracket-tools-cache/src/memory_storage.rs` — `MemoryStorage` (HashMap in `Mutex`, in-process, no persistence; for tests/TUI), `new()`/`Default`
- `crates/bracket-tools-cache/tests/storage_tests.rs` — Integration tests for NullStorage, SledStorage, MemoryStorage (Sled/Memory share generic `check_*<S: Storage>` helpers + thin per-backend wrappers)

**Status:** Storage trait and implementations complete. No `Send` bound on trait futures (sled `Db` is `!Sync`).

## bracket-tools-startgg-schema

cynic codegen types from the start.gg GraphQL schema.

**Key files:**
- `crates/bracket-tools-startgg-schema/src/schema.rs` — cynic schema registration
- `crates/bracket-tools-startgg-schema/src/scalars.rs` — shared scalars: `Timestamp(pub i64)` (API returns JSON numbers) + tolerant `Id` (accepts numeric AND string JSON ids — live API sends numbers, `preview_*` ids are strings; registered against the built-in ID TypeLock `cynic::Id` via manual impl_scalar expansion). Older modules' `Timestamp(pub String)` + `cynic::Id` remain (multiple Rust types may map to one GraphQL scalar) but `cynic::Id` fields FAIL on live numeric ids — migrate old modules to `scalars::Id` when they get live use
- `crates/bracket-tools-startgg-schema/src/enums.rs` — shared `ActivityState`, `BracketType` cynic enums
- `crates/bracket-tools-startgg-schema/src/get_sets_for_event.rs` — THE scheduler read query: event's full set list, `hideEmpty: false` (keeps unfilled future sets — their prereq edges are the bracket DAG spine), `sortType: ROUND`; Set subtree derives Clone
- `crates/bracket-tools-startgg-schema/src/get_event_structure.rs` — event skeleton: phases (state), phase groups (bracketType, numRounds, wave, rounds), tournament id/slug, numEntrants
- `crates/bracket-tools-startgg-schema/src/mark_set_called.rs` / `mark_set_in_progress.rs` — the repo's first mutations (`MutationBuilder`); 4-field Set payload (id/state/startedAt/completedAt)
- `crates/bracket-tools-startgg-schema/src/report_bracket_set.rs` — set-result mutation (`reportBracketSet`: winnerId, isDQ, per-game `BracketSetGameDataInput` with character `selections`); returns the affected sets as the shared 4-field payload
- `crates/bracket-tools-startgg-schema/src/get_event_characters.rs` — event → videogame → character roster (the reporting vocabulary)
- `crates/bracket-tools-startgg-schema/src/get_events_for_tournament.rs` — tournament slug → its full event list (id/slug/name + `videogame { id name }`; the smoke bin's `--tournament` expansion and `--init-tournament`'s source)
- `crates/bracket-tools-startgg-schema/src/admin_probe.rs` — preflight admin probe: one query answering `currentUser { id }` + `tournament(id:) { admins { id } }` (admin-only field; hidden for non-admin tokens — absence is signal). Uses tolerant `scalars::Id`
- `crates/bracket-tools-startgg-schema/src/get_tournament_for_id.rs` — Tournament + paginated participants query (`ParticipantConnection` has `pageInfo { totalPages }` + `nodes`)
- `crates/bracket-tools-startgg-schema/src/get_games_for_set.rs` — Set + games + selections query (largest); Entrant includes `id` field
- `crates/bracket-tools-startgg-schema/src/get_player_for_player_id.rs`
- `crates/bracket-tools-startgg-schema/src/get_user_for_user_slug.rs`
- `crates/bracket-tools-startgg-schema/src/get_tourney_id.rs`
- `crates/bracket-tools-startgg-schema/src/get_sets_of_player_after_datetime.rs`

**Build:** `build.rs` reads `gg_info/schema.graphql` via cynic_codegen.

## Conversion Pipeline

```
API query -> cynic response -> QueryResult{id, response}
  -> TryFrom -> HydratedGg* (gg_data_types)        # player, set
  (tournament: fetch_all_pages -> extract_tournament_participants_page
     + tournament_name -> HydratedGgTournament)    # paginated path, no QueryResult
  -> Normalizable -> core types (Tournament, Set, etc.)
  -> Dehydrateable -> DehydratedGg* (for cache storage)
```

## Data Flow (SDK layer)

Three cache tiers, checked in order on a lazy handle's `.get()`:

```
LazyTournament/LazySet/LazyPlayer.get()
  -> GgSession in-memory tier (typed Arc<HydratedGg*>, per-id OnceCell; coalesces misses)
       -> GGProvider.get_*()
            -> sled Storage (bincode bytes) — hit: deserialize & return
            -> miss: governor rate limit -> reqwest HTTP POST to api.start.gg/gql/alpha
                     -> deserialize cynic response -> TryFrom -> HydratedGg*
                     -> store in sled, return
  -> store Arc in session map, return Arc to caller
```

`GGProvider.get_*()` can still be called directly (without a session) for the
eager, two-tier (sled -> HTTP) path. The session adds the in-memory tier and
hands back `Arc<HydratedGg*>` shared across handles. Navigation
(`participants()`, `players()`) fetches the parent once and yields more lazy
handles rather than eager fetches.
